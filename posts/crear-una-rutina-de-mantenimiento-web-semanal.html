Una web también se abandona de a poquito

No siempre cae de golpe. A veces se llena de enlaces viejos, imágenes pesadas, textos vencidos y detalles que nadie mira.

Por ASPF · Actualizado 12/06/2026 · 8–11 min
WhatsAppFacebookX

Una web puede seguir online y, aun así, estar descuidada. Carga, pero cada vez un poco peor. Tiene artículos, pero algunos ya no dicen la verdad. Enlaza, pero algunas rutas murieron. Se ve bien en escritorio, pero en móvil algo salta.

El mantenimiento semanal existe para evitar esa degradación lenta.

No es una lista infinita. Es una vuelta corta por las zonas que más sostienen el sitio.

Primero mirar la portada como visitante

Abrí la home sin entrar como dueño. ¿Se entiende qué es el sitio? ¿Las categorías tienen sentido? ¿Los últimos artículos aparecen bien? ¿Algo tarda demasiado? ¿El móvil se lee sin pelear?

La portada es la puerta. Si esa puerta se volvió confusa, todo lo demás sufre.

Después enlaces

Un enlace roto parece pequeño hasta que corta un recorrido. Revisá los enlaces internos más importantes: artículos relacionados, categorías, legales, contacto, sitemap si corresponde.

No hace falta auditar todo cada semana. Pero sí conviene tocar una zona y dejarla mejor que antes.

Imágenes y peso

Una imagen subida sin preparar puede arruinar la carga de una página. Revisá si las últimas publicaciones tienen imágenes razonables, dimensiones declaradas, nombre claro y alt útil.

En crear imágenes web limpias y livianas esa revisión se vuelve parte del taller.

Contenido viejo con fecha vencida

Algunas páginas necesitan actualización. No por moda, sino porque cambió una herramienta, murió una fuente, apareció una consulta nueva o el artículo quedó demasiado genérico.

Una rutina semanal puede elegir solo una pieza vieja y mejorarla.

Revisar señales sin obsesión

Search Console, analítica, errores de indexación o páginas con muchas impresiones pueden orientar. Pero la rutina no debería convertirse en mirar números por mirar.

La salida tiene que ser una acción: corregir, enlazar, actualizar, borrar ruido, preparar una próxima mejora.

Guardar el cambio

Si usás Git, cada mantenimiento importante merece commit claro. No “actualizaciones varias”, sino qué se tocó: enlaces, imagen, portada, artículo viejo, sitemap.

Ese registro evita repetir trabajo y ayuda a entender cuándo empezó un problema.

Una rutina de media hora

Una buena sesión puede ser breve: portada, una página, enlaces, una imagen, un dato técnico y una nota de próxima acción.

El mantenimiento sirve cuando se puede sostener. Si se vuelve enorme, se abandona.

Dudas para mantener sin volverte loco

¿Hay que revisar todo cada semana?

No. Conviene revisar una zona clara y dejar una mejora concreta.

¿Qué miro primero?

Home, móvil, enlaces principales, últimas publicaciones e imágenes pesadas.

¿Cómo sé si la rutina sirve?

Si cada semana deja una mejora registrada y una próxima acción clara.
