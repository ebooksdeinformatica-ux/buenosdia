Cómo saber si un artículo merece publicarse o conviene reescribirlo

No todo texto terminado merece publicarse. A veces el mejor movimiento no es subirlo, sino frenarlo, reescribirlo, fusionarlo o convertirlo en una sección de una guía más fuerte.

Publicado: 2026-07-02 · Autor: ASPF · Lectura: 14 min

Publicar por publicar rompe un sitio desde adentro. Al principio parece avance: más URLs, más tarjetas, más sitemap, más movimiento. Pero si las piezas son débiles, repetidas o demasiado parecidas, el sitio empieza a perder forma. El lector lo nota. Google también.

Un artículo puede estar correctamente escrito y aun así no ser suficiente. Puede tener título, descripción, subtítulos, enlaces internos y hasta buena ortografía. Pero si no resuelve una intención real, si repite lo que ya dijiste o si parece una ficha generada para llenar espacio, conviene frenarlo.

Esta guía sirve como filtro. Antes de publicar, el artículo tiene que pasar por una revisión editorial real. No una revisión decorativa. Una revisión incómoda. De esas que hacen mejores las páginas.

Índice de revisión

  1. La intención se entiende en una frase
  2. El artículo tiene una diferencia real
  3. Tiene profundidad suficiente
  4. Incluye ejemplos o casos concretos
  5. La estructura no parece plantilla
  6. Publicar, reescribir, fusionar o noindex
  7. Checklist final

1. La intención se entiende en una frase

La primera pregunta es simple: ¿para qué existe este artículo?

Si la respuesta es “para tener más contenido”, el artículo no está listo. Si la respuesta es “porque este tema puede traer visitas”, tampoco alcanza. Una página necesita una intención más precisa.

Una intención fuerte suena así: “ayudar a alguien que tiene un borrador y no sabe si publicarlo o reescribirlo”. Esa frase define el problema, el lector y la utilidad.

Una intención débil suena así: “hablar sobre redacción SEO”. Es demasiado amplia. Podría convertirse en mil cosas. Cuando la intención es blanda, el texto suele salir blando.

Antes de publicar, escribí la intención en una línea. Si no sale, reescribí. Si sale pero el artículo no la cumple, también reescribí.

2. El artículo tiene una diferencia real

Un sitio no crece solo por tener muchas páginas. Crece cuando cada página aporta un ángulo, una utilidad o una experiencia que no estaba cubierta igual en otra parte del sitio.

Si ya tenés una guía sobre publicar una página simple, no conviene crear cinco artículos casi iguales sobre título, descripción, primera pantalla e índice, salvo que cada uno tenga profundidad y una razón clara. Si no, esos artículos deben ser satélites o secciones, no piezas principales.

La pregunta brutal es: si cambio el título de este artículo, ¿seguiría sonando igual que otro post?

Si la respuesta es sí, hay problema. Eso significa que el texto no tiene personalidad editorial ni utilidad específica. Está ocupando espacio.

Un artículo merece publicarse cuando tiene una función propia dentro del sitio. Puede ser una guía pilar, una respuesta puntual, una comparación, un caso práctico, una lista curada o una reflexión fuerte. Pero tiene que ser algo.

3. Tiene profundidad suficiente

Profundidad no significa escribir 3000 palabras siempre. Significa cumplir la promesa con el nivel de detalle que el lector necesita.

Un texto de 800 palabras puede ser profundo si resuelve una duda pequeña con precisión. Un texto de 2500 palabras puede ser flaco si repite ideas generales. La cantidad no salva una página vacía.

Hay señales claras de poca profundidad:

  • muchos subtítulos y pocas ideas reales;
  • frases que podrían estar en cualquier blog;
  • cero ejemplos;
  • consejos obvios sin explicación;
  • un cierre que no cambia nada;
  • enlaces internos puestos solo para decorar.

Una página profunda hace avanzar al lector. Después de leerla, la persona entiende mejor, decide mejor o puede hacer algo concreto.

Si el artículo no deja ninguna herramienta mental o práctica, todavía no merece publicarse.

4. Incluye ejemplos o casos concretos

Los ejemplos son una prueba de verdad. Cuando un artículo no puede mostrar un caso concreto, suele ser porque está flotando.

Ejemplo débil:

“Es importante escribir contenido de calidad y revisar antes de publicar.”

Eso dice algo correcto, pero no ayuda demasiado.

Ejemplo más útil:

Borrador débil: “Cómo mejorar una página web”.

Problema: título demasiado amplio, promesa genérica, sin lector definido.

Versión mejor: “Cómo revisar la primera pantalla de una página simple antes de publicarla”.

Por qué mejora: define una parte concreta de la página, una acción y un momento de uso.

El ejemplo obliga a pensar. También le da al lector una escena. Ya no está leyendo humo. Está viendo una decisión editorial.

Un buen artículo no necesita estar lleno de ejemplos, pero debería tener al menos uno que aterrice la idea.

5. La estructura no parece plantilla

Una estructura ayuda. Una plantilla repetida mata. Si cada artículo del sitio tiene la misma forma, el lector siente que cambian las palabras pero no cambia la experiencia.

El problema no es usar subtítulos, listas o checklist. El problema es usarlos sin necesidad. Cada artículo tiene que encontrar su forma según la intención.

Una guía práctica puede tener índice, pasos, ejemplo y checklist. Una reflexión puede necesitar menos estructura y más voz. Una comparación necesita criterios claros. Un tutorial necesita secuencia. Una página pilar necesita jerarquía.

Si la estructura aparece antes que la idea, el artículo queda mecánico.

La prueba es leer solo los subtítulos. ¿Cuentan una historia útil? ¿O parecen cajas vacías? Si los subtítulos no muestran avance, el contenido probablemente tampoco.

6. Publicar, reescribir, fusionar o noindex

No todos los textos tienen el mismo destino. Hay cuatro decisiones posibles.

Publicar

Publicá cuando el artículo tiene intención clara, diferencia real, desarrollo suficiente, ejemplos, enlaces internos lógicos y una razón para existir dentro del sitio.

Reescribir

Reescribí cuando la idea es buena pero el texto está flojo. Tal vez falta ejemplo, falta foco, sobra introducción, el título promete mal o la estructura se siente genérica.

Fusionar

Fusioná cuando el artículo no aguanta solo, pero sirve como parte de una guía mayor. Esto es clave. Hay temas que no merecen una URL propia, pero sí una sección fuerte dentro de una guía pilar.

Noindex o sacar de foco

Usá noindex o sacalo del foco cuando la pieza existe, pero no conviene empujarla como resultado orgánico. Puede servir internamente, como apoyo, archivo o prueba, pero no tiene que competir en Google.

La madurez editorial aparece cuando dejás de tratar todas las URLs como si fueran igual de importantes.

7. Checklist final antes de publicar

  • ¿La intención del artículo se puede escribir en una frase?
  • ¿La página responde mejor que un texto genérico?
  • ¿Tiene una diferencia clara frente a otros posts del sitio?
  • ¿Incluye al menos un ejemplo concreto?
  • ¿Cada sección aporta algo nuevo?
  • ¿La introducción entra rápido al problema?
  • ¿El título promete algo que el texto cumple?
  • ¿La descripción resume valor real y no relleno?
  • ¿Los enlaces internos tienen sentido?
  • ¿El artículo debería ser URL propia o sección de una guía pilar?

Conclusión: publicar menos basura también es SEO

La poda también es trabajo editorial. Frenar un artículo débil no es perder tiempo. Es evitar que el sitio se llene de páginas sin peso.

Una web que quiere crecer necesita criterio. No cada borrador merece indexarse. No cada idea merece URL propia. No cada texto terminado merece entrar al sitemap.

Un artículo merece publicarse cuando mejora el sitio. Si solo aumenta el contador, conviene reescribirlo.

La próxima vez que un texto parezca listo, no preguntes solo si está terminado. Preguntá si merece ocupar lugar. Esa diferencia puede salvar una web entera.
