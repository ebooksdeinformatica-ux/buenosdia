La imagen que subís sin pensar puede ser el ancla de toda la página

No se trata de subir algo lindo. Se trata de que la imagen tenga función, pese poco y no rompa la lectura.

Por ASPF · Actualizado 12/06/2026 · 9–12 min
WhatsAppFacebookX

Una página puede estar bien escrita y aun así sentirse pesada. Abrís, esperás, el texto salta, la portada tarda, el móvil se queda pensando. Después mirás y el culpable era una imagen subida como venía: enorme, con nombre raro, sin tamaño declarado y sin una razón clara para estar ahí.

La imagen parece detalle menor hasta que frena todo.

En un blog liviano, cada imagen tiene que ganarse su lugar. No por bonita. Por útil.

Antes del formato viene la función

La primera pregunta no es PNG, JPG o WebP. La primera pregunta es: ¿qué hace esta imagen por el artículo?

Puede orientar. Puede mostrar una captura. Puede explicar una relación. Puede funcionar como portada social. Puede dar descanso visual. Pero si no hace nada de eso, probablemente esté rellenando.

El original no es la versión web

Guardar el original está bien. Publicarlo tal cual casi nunca.

La versión web debería ser una copia pensada para cargar rápido: tamaño razonable, peso controlado y calidad suficiente para la pantalla donde se va a ver. No hace falta subir una imagen gigante para mostrarla chica.

Esto se nota mucho en webs simples, donde una sola imagen pesada puede arruinar una página que por HTML era rápida.

El nombre también ordena

Un archivo llamado IMG_3847-final-final.png no ayuda a nadie. Ni al sitio ni a vos cuando vuelvas dentro de tres meses.

Un nombre claro como imagen-web-liviana-portada.webp ya dice qué es. En minúsculas, sin espacios raros, con guiones. Parece una manía, pero es memoria de proyecto.

Alt sin piloto automático

El texto alternativo no tiene que repetir el título como loro. Tiene que describir lo importante de la imagen para quien no la ve o para cuando el contexto necesita una explicación.

Si es una captura, contá qué muestra. Si es una portada conceptual, describí la idea. Si no aporta contenido y es decorativa, tratala como corresponde.

La imagen social es otro trabajo

La portada que aparece al compartir en WhatsApp, Facebook o X no siempre cumple la misma función que una imagen dentro del artículo. Tiene que entenderse rápido, tener proporción correcta y no depender de detalles minúsculos.

Para muchos casos, 1200×675 funciona bien como punto de partida en formato amplio, siempre cuidando peso final.

Una revisión rápida antes de publicar

Abrí la página en móvil. Mirá si salta. Mirá si la imagen tarda. Mirá si empuja el texto. Mirá si el alt tiene sentido. Mirá si el archivo pesa más de lo necesario.

Si la imagen no mejora la lectura, no la defiendas por costumbre.

En hacer una web estática que cargue rápido esta decisión se vuelve parte del rendimiento general.

Preguntas para no subir peso inútil

¿La imagen explica algo?

Si no orienta, muestra, aclara o acompaña una idea, tal vez sobra.

¿Estoy publicando el original sin necesidad?

Guardá el original aparte y subí una copia preparada para web.

¿El nombre del archivo se entiende?

Si no se entiende ahora, menos se va a entender cuando el sitio crezca.
