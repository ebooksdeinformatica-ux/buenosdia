Una imagen web tiene que ayudar, no hacer pesada la página

El objetivo no es subir la foto más grande posible. Es mostrar lo necesario, con peso bajo y contexto claro.

Por ASPF · Actualizado 12/06/2026 · 9–12 min
WhatsAppFacebookX

Una imagen puede hacer que una página parezca más trabajada. También puede arruinarla si pesa demasiado, empuja el texto, se ve borrosa o no tiene ninguna relación con lo que se está leyendo.

En una web liviana, la imagen tiene una función: orientar, explicar, acompañar una idea o servir como portada para compartir. Si solo está para rellenar, conviene sacarla o mejorarla.

Primero decidí para qué existe

No todas las imágenes cumplen el mismo trabajo. Una portada social necesita verse bien en vista previa. Una captura técnica debe mostrar un detalle. Un diagrama debe aclarar una relación. Una imagen decorativa no debería competir con la lectura.

Cuando definís función, definís tamaño, formato y texto alternativo.

Tamaño antes de compresión

Comprimir una imagen gigante ayuda, pero no resuelve todo. Si la página va a mostrarla a 1200 píxeles de ancho, no tiene sentido publicar una versión enorme sin necesidad.

Guardá el original aparte y exportá una copia para web. Así podés volver atrás sin destruir el archivo de mayor calidad.

Formato según uso

JPEG puede funcionar bien para fotos. PNG sirve cuando hay transparencia o gráficos simples, aunque puede pesar más. WebP suele dar buen equilibrio para muchas imágenes web modernas.

No elijas formato por moda. Compará calidad visible y peso final.

Nombres de archivo claros

IMG_20260612_0031.png no explica nada. Un nombre como web-estatica-imagen-optimizada.webp ayuda a ordenar archivos y entender el recurso meses después.

Usá minúsculas, guiones y palabras simples. Evitá acentos y espacios en nombres de archivo.

Alt que describa

El texto alternativo no debe repetir el título del artículo como robot. Tiene que describir lo importante de la imagen. Si la imagen es decorativa, el tratamiento puede ser distinto según el caso.

Para una captura, explicá qué se ve. Para un diagrama, resumí la relación. Para una portada, describí el concepto.

Dimensiones declaradas

Cuando indicás ancho y alto, el navegador puede reservar espacio antes de cargar. Eso reduce saltos visuales y mejora la lectura.

Una página que se mueve mientras leés parece improvisada, aunque el texto sea bueno.

Proceso mínimo

  1. Guardar original.
  2. Crear copia para web.
  3. Redimensionar.
  4. Comprimir.
  5. Nombrar claro.
  6. Escribir alt.
  7. Probar en móvil.

En hacer una web estática que cargue rápido la imagen aparece como una de las capas principales de rendimiento.

Evitar derechos dudosos

No uses imágenes tomadas al azar. Usá material propio, recursos con licencia clara o gráficos generados específicamente para el sitio. Guardá referencia de origen cuando corresponda.

Una imagen “gratis” sin revisar puede traer problemas después.

Preguntas frecuentes

¿Qué tamaño uso para portada?

Para compartir, 1200×675 funciona bien en formato 16:9. Ajustá según diseño y peso final.

¿WebP sirve siempre?

Suele ser una buena opción, pero conviene probar compatibilidad y calidad visible según el caso.

¿Tengo que poner alt en todas?

Sí, pero el tipo de alt depende de la función. Debe ayudar a entender, no rellenar.

¿Cuándo una imagen pesa demasiado?

Cuando retrasa la carga sin aportar valor equivalente. Compará peso, tamaño visible y utilidad real.
