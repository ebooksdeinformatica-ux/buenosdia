No necesitás una computadora perfecta para crear un blog serio

Necesitás una idea clara, archivos que puedas mantener y una forma de publicar que no se coma toda tu energía.

Por ASPF · Actualizado 12/06/2026 · 10–13 min
WhatsAppFacebookX

La computadora perfecta es una excusa elegante. Falta memoria, falta un monitor, falta otro programa, falta estudiar una tecnología más. Mientras tanto, el blog sigue sin una sola publicación que alguien pueda leer.

Un blog serio no nace de la potencia del equipo. Nace de una decisión editorial, una estructura que puedas sostener y una forma simple de publicar. Una notebook común puede escribir texto, editar HTML, preparar una imagen y subir archivos. Para empezar, eso alcanza.

Yo viví internet cuando crear una web exigía resolver mucho con menos herramientas que ahora. También vi proyectos llenos de efectos que no tenían nada para decir. El equipo importa, pero una persona vuelve por lo que encuentra, no por la cantidad de memoria RAM con la que fue escrito.

Primero definí por qué debería existir

“Un blog de tecnología” es demasiado amplio. “Un blog para reconstruir proyectos digitales con pocos recursos y experiencia real” ya tiene dirección. Antes del diseño, respondé:

  • ¿A quién le hablás?
  • ¿Qué problema entiende mejor después de leerte?
  • ¿Qué experiencia propia podés aportar?
  • ¿Qué temas no vas a publicar aunque tengan búsquedas?
  • ¿Qué ritmo podés sostener durante meses?

Google recomienda contenido útil, confiable y pensado principalmente para personas. Eso no elimina el SEO; evita que el buscador sea la única razón por la que una página existe.

Una estructura mínima que pueda crecer

No necesitás veinte plugins. Un blog estático puede empezar con una portada, una carpeta de publicaciones, categorías, estilos, imágenes, un archivo de datos, robots.txt, sitemap.xml y una página 404.

GitHub Pages aloja sitios estáticos tomando HTML, CSS y JavaScript desde un repositorio, y permite usar un dominio personalizado. La computadora local solo edita y envía cambios: no tiene que actuar como servidor.

Una estructura posible es:

/
├── index.html
├── posts/
├── categorias/
├── assets/css/
├── assets/img/
├── data/posts.json
├── robots.txt
├── sitemap.xml
└── 404.html

Qué necesita cada publicación

Una página no mejora por completar campos SEO si el cuerpo no responde nada. Pero la parte técnica tampoco debe quedar librada al azar. Cada artículo necesita un title específico, H1 claro, descripción única, URL estable, canonical, autor, fecha real, imagen optimizada, datos estructurados y enlaces internos.

Después viene lo difícil: una apertura propia, ejemplos reales, límites, decisiones y una razón para que ese artículo no pueda intercambiarse con otro cambiando solamente el título.

Automatizar el marco, no el pensamiento

La automatización puede generar sitemap, actualizar categorías, comprobar enlaces, insertar componentes comunes y detectar frases repetidas. Eso es bueno.

Lo que no sirve es producir cincuenta textos con la misma introducción, el mismo plan de siete días y el mismo cierre. El generador puede construir archivos. No puede decidir por sí solo qué experiencia merece ser contada.

Buenosdia.com tuvo que reconocer ese problema: había cientos de páginas, pero muchas eran el mismo molde. Corregirlas significa rehacer la intención y el contenido, no cambiar dos frases.

Trabajar con pocos recursos

En una computadora limitada, separá las etapas. Primero investigá y guardá fuentes. Después escribí con pocas pestañas. En otra sesión corregí, prepará la imagen y publicá. La multitarea consume memoria de la máquina y de la cabeza.

  1. Definí una entrega.
  2. Escribí sin editar imágenes al mismo tiempo.
  3. Revisá afirmaciones y enlaces.
  4. Comprimí una imagen adecuada.
  5. Actualizá el catálogo y el sitemap.
  6. Probá el resultado en móvil y escritorio.

Para aprovechar mejor una máquina limitada, seguí con cómo usar una computadora vieja para crear valor digital.

Velocidad antes que decoración

Una imagen enorme puede destruir la ventaja de un sitio estático. Usá dimensiones razonables, compresión y formatos adecuados. Declarar ancho y alto evita movimientos durante la carga. El texto alternativo debe describir la imagen, no repetir palabras clave.

También revisá fuentes externas, scripts y efectos. Cada elemento tiene que justificar el peso y el mantenimiento que agrega.

SEO sin vivir persiguiendo trucos

El SEO técnico es un piso. Los buscadores deben encontrar y entender el contenido, pero una página indexada que no ayuda a nadie sigue siendo mala.

Trabajá con una intención real. Si alguien busca cómo crear un blog con una notebook vieja, necesita saber qué estructura alcanza, cómo publicar, qué límites existen y cómo evitar que la herramienta se convierta en excusa. No necesita leer veinte veces la misma frase exacta.

Un ritmo que no destruya la calidad

No publiques todos los días si cada página baja el nivel del sitio. Al principio puede funcionar mejor una publicación profunda, una actualización de contenido viejo y una revisión técnica por semana. Acelerá cuando el sistema demuestre calidad, no cuando el generador demuestre velocidad.

Cómo empezar

  1. Escribí una frase que explique para quién existe el blog.
  2. Elegí pocas categorías claras.
  3. Creá un repositorio y una portada simple.
  4. Publicá un artículo que resuelva una necesidad real.
  5. Agregá metadatos, enlaces e imagen.
  6. Probá y corregí.

El primer blog no tiene que parecer una empresa gigante. Tiene que funcionar, poder mantenerse y contener algo que valga la pena leer. Después crecerá.

Preguntas frecuentes

¿Necesito WordPress?

No. Es una opción, pero también podés usar un sitio estático o un generador. La elección depende de funciones, mantenimiento y conocimientos.

¿GitHub Pages acepta dominio propio?

Sí. Permite publicar un sitio estático y configurarlo con un dominio personalizado.

¿Cuántos artículos necesito para lanzar?

No hay un número mágico. Pocas publicaciones excelentes aportan más que decenas de textos repetidos.

¿Cuándo conviene mejorar la computadora?

Cuando el límite está medido: bloqueos, sistema inseguro o entregas retrasadas por hardware. No porque una publicidad te hizo sentir atrasado.
