# Control de variacion editorial

Objetivo: evitar que las publicaciones salgan con la misma arquitectura aunque cambie el tema.

## Regla base

Cada nueva publicacion debe tener una decision formal propia antes de escribirse.

No alcanza con elegir tema, keyword o categoria.
Tambien hay que elegir forma.

## Comparacion obligatoria

Antes de publicar, comparar la pieza nueva con las tres anteriores.

Revisar:

- apertura;
- ritmo;
- cantidad de subtitulos;
- longitud de secciones;
- tipo de ejemplos;
- relacion entre escena y explicacion;
- tipo de cierre;
- tono de autoridad;
- nivel practico o narrativo.

## Rotacion de formas

Alternar familias de pieza:

- cronica;
- ensayo;
- guia;
- carta abierta;
- lista comentada;
- historia breve;
- diario de observacion;
- conversacion interna;
- desmontaje de una idea;
- mapa practico;
- escena unica expandida;
- manifiesto breve.

## Uso del radar externo

El radar no debe dictar el molde.

Cada consulta externa debe usarse para una sola funcion:

- titulo;
- estructura;
- experiencia de lectura;
- densidad practica;
- navegacion;
- claridad;
- ritmo.

No usar siempre la misma funcion.
No mirar siempre las mismas fuentes.

## Prueba final

Si el texto parece una version de otro post con diferente titulo, vuelve a taller.

Si suena correcto pero generico, vuelve a taller.

Si la version internacional suena traducida y sin voz, vuelve a taller.
