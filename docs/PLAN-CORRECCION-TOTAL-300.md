# PLAN DE CORRECCIÓN TOTAL — 300 PUBLICACIONES

## Alcance real

La web contiene 300 publicaciones distribuidas en seis categorías. Todas las publicaciones existentes deben revisarse y, salvo que una pase una revisión editorial excepcional, se consideran borradores defectuosos que deben reescribirse por completo.

No se trata de corregir solamente Tecnología. El trabajo obligatorio incluye:

1. Tecnología — 50 publicaciones.
2. Pan y Circo — 50 publicaciones.
3. Alimentación — 50 publicaciones.
4. Deportes — 50 publicaciones.
5. Matrix — 50 publicaciones.
6. Saliendo de la Matrix — 50 publicaciones.

Total: 300 publicaciones.

## Norma obligatoria

Todo se rehace bajo:

- ACTIVAR MODELO DE BLOG GITHUB;
- MODELO MAESTRO DE BUENOSDIA.COM;
- 📚 BIBLIOTECA PDF conversacional;
- voz ASPF;
- fuentes verificables cuando corresponda;
- auditoría antiplantilla y revisión humana.

## Qué significa corregir

Corregir no significa cambiar la introducción o sustituir algunas frases. Significa revisar y, cuando sea necesario, reemplazar completamente:

- intención del artículo;
- título SEO y H1;
- descripción;
- apertura;
- desarrollo;
- ejemplos;
- pasos y decisiones;
- riesgos y límites;
- fuentes;
- preguntas frecuentes;
- enlaces internos;
- etiquetas;
- imagen;
- metadatos y datos estructurados.

## Conservación de URLs

Las URLs útiles se mantienen para evitar enlaces rotos. Una URL solo cambia si el tema es irreparable, engañoso o está duplicado. Todo cambio de URL debe incluir redirección.

## Ritmo seguro

Cada lote de 50 se trabaja en microbloques de 5 o 10 artículos. Un microbloque solo se publica cuando todos sus artículos aprobaron:

- contenido real y específico;
- voz editorial;
- verificación de afirmaciones;
- SEO técnico;
- enlaces internos;
- imagen y accesibilidad;
- auditoría automática;
- revisión final.
