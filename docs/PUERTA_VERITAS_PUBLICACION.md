# Puerta VERITAS de publicacion

Este documento existe para evitar que el proyecto publique piezas correctas pero flojas.

## Regla central

Ninguna publicacion nueva cuenta como terminada solo porque tenga HTML, meta, sitemap o categoria.

Una publicacion cuenta como terminada cuando merece existir para el lector.

## Antes de escribir

THOT debe confirmar:

- intencion de busqueda clara;
- diferencia real frente a los ultimos posts;
- formato elegido por utilidad, no por costumbre;
- lectura de radar externo o criterio equivalente actualizado;
- profundidad minima necesaria para que el usuario no sienta una pieza liviana;
- enlaces internos utiles previstos;
- categoria correcta;
- riesgo de canibalizacion revisado.

MNEMOSINE debe confirmar:

- lugar dentro del mapa conceptual;
- relacion con publicaciones existentes;
- hueco que cubre;
- piezas cercanas para enlazar;
- si la idea ya fue repetida con otro nombre.

MORFEO debe confirmar:

- recorrido lector despues del articulo;
- bloque relacionado util;
- salida hacia categoria, ruta o lectura complementaria;
- que el post no termine en callejon.

LEONARDO debe confirmar:

- legibilidad mobile;
- jerarquia visual;
- que no parezca tarjeta generica;
- que el diseño sostenga la lectura.

## Durante la escritura

No se publica si ocurre cualquiera de estos sintomas:

- el texto parece relleno correcto;
- las secciones son demasiado cortas sin razon;
- todos los H2 tienen el mismo ritmo;
- el cierre suena automatico;
- se repite el molde de los ultimos posts;
- hay consejos obvios sin desarrollo;
- falta escena, ejemplo, contraste o utilidad concreta;
- el lector no aprende nada nuevo;
- el articulo podria cambiar de titulo y seguir igual.

## Idiomas

EN y FR no son resumenes.

Una familia internacional solo se aprueba si:

- ES esta completo;
- EN conserva profundidad proporcional;
- FR conserva profundidad proporcional;
- los subtitulos principales estan representados;
- los enlaces internos equivalentes funcionan;
- el tono humano sigue vivo;
- `data/i18n_posts.json` y `sitemap-i18n.xml` solo incluyen familias aprobadas.

## Despues de escribir

VERITAS revisa:

- post ES creado;
- datos actualizados;
- categoria actualizada;
- home actualizada si corresponde;
- sitemap principal actualizado;
- idiomas aprobados o marcados como pendientes;
- sitemap internacional limpio;
- sin rutas falsas;
- sin duplicados;
- sin prometer calidad que no existe.

## Decision final

Si una pieza no pasa esta puerta, no se publica.

Si ya se publico pero no cumple, se marca como pendiente, se retira del indice internacional si corresponde y se repara antes de acelerar nuevas piezas.
